# 作业题目：ImageLab - PIL 图像变换工具实现与验收

## 一、问题情境

你正在开发一个命令行图像处理工具。用户希望读取图片、保存图片，并对图片进行放大、缩小、旋转、裁剪、反色、模糊、边缘提取和中值滤波等操作。

本作业要求你提交终版代码和报告。系统会通过隐藏验收检查代码行为，并通过短答辩判断你是否理解图像对象、像素处理、参数边界和验证方法。

## 二、任务目标

请使用 Python 和 Pillow/PIL 实现一个命令行图像变换工具 `ImageLab`。程序入口固定为：

```text
final/image_ops.py
```

你的最终版本应支持读取、保存、放大、缩小、旋转、裁剪、反色、均值模糊、固定卷积核边缘提取和中值滤波，并且能对非法参数和损坏图片给出明确错误。

## 三、功能要求

- 读取图片并转换成稳定可处理的图像模式
- 保存图片到目标路径
- 按比例放大和缩小图片
- 按角度旋转图片
- 按坐标裁剪图片
- 反色处理
- 均值模糊
- 固定卷积核边缘提取
- 中值滤波
- 命令行入口，能从输入路径生成输出图片
- 对非法缩放比例、非法裁剪框、非法滤波尺寸、未知操作给出明确错误
- 将业务逻辑拆分为可被系统调用的函数，不能全部写在命令行解析里

以上功能全部属于必做要求，不能作为 bonus 申报。

## 四、固定函数接口

`final/image_ops.py` 除了提供命令行入口外，必须暴露以下可调用函数。函数名不能修改。

- `load_image(path)`
- `save_image(image, path)`
- `resize_image(image, scale)`
- `rotate_image(image, angle)`
- `crop_image(image, left, top, right, bottom)`
- `invert_image(image)`
- `blur_image(image)`
- `edge_detect(image)`
- `median_filter(image, size=3)`
- `transform_image(input_path, output_path, operation, **kwargs)`

接口约束：

- 图像处理函数应返回新的 Pillow `Image` 对象，不能依赖全局状态。
- `load_image` 应返回可稳定处理的图像对象，例如 RGB 图像。
- `save_image` 必须实际写出目标文件，并返回保存路径或目标路径字符串。
- `resize_image` 的 `scale` 必须大于 0。
- `crop_image` 必须校验裁剪框在图像范围内，且 `right > left`、`bottom > top`。
- `median_filter` 的 `size` 必须为大于 0 的奇数。
- `transform_image` 应作为统一调度入口，至少支持 `resize`、`rotate`、`crop`、`invert`、`blur`、`edge`、`median`。
- 非法参数、未知操作、无法读取图片时必须抛出明确异常，不能静默忽略。

## 五、报告要求

`report/report.md` 必须包含：

- 项目结构和运行方式
- 核心图像处理函数和模块划分
- 关键功能实现说明，尤其是图像模式、裁剪校验、均值模糊、边缘检测和中值滤波
- 自测输入、预期结果和验证方法
- 自己的关键实现说明和工程取舍
- 最终反思和已知不足

报告中的关键说法应能对应到具体函数、行号、隐藏验收项或运行结果。

## 六、自主扩展 Bonus

在完成全部必做功能的前提下，学生可以自由设计并实现额外图像功能。只有学生自定义扩展功能可以作为 bonus；上述必做功能都不能算作 bonus。

可选扩展示例：

- 灰度化、亮度/对比度调整
- 自定义卷积核
- 批量处理目录图片
- 添加水印或文字
- 直方图统计
- 多步流水线变换
- 更友好的交互式命令行模式

扩展功能必须在 README 和报告中说明设计动机、实现方法和验证方式。必做要求缺失时，bonus 不用于弥补必做得分。

- 学生是否能说明自测输入、预期结果和边界情况

- 学生是否能把自己的实现说明对应到终版代码、验收要求和报告证据
