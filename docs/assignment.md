# 作业题目：TaskFlow - 使用 AI 辅助实现命令行任务管理器

## 一、作业背景

在 AI 辅助编程已经普及的背景下，本作业不禁止学生使用 AI，而是要求学生展示如何负责任地使用 AI：能提出明确需求，能发现 AI 生成代码中的问题，能通过测试验证修改，能说明哪些工作来自自己的判断。

## 二、任务描述

请使用 Python 实现一个命令行任务管理器 `TaskFlow`。学生可以使用 AI 辅助完成，但必须保留 AI 协作过程，并对最终代码负责。

程序入口固定为：

```text
final/taskflow.py
```

## 三、功能要求

任务字段至少包含：

- `id`：任务唯一编号
- `title`：任务标题
- `priority`：任务优先级，只能是 `high`、`medium`、`low`
- `deadline`：截止日期，格式为 `YYYY-MM-DD`
- `status`：任务状态，至少包含 `pending` 和 `completed`

命令行功能要求：

- 添加任务
- 删除任务
- 完成任务
- 查看任务
- 按状态筛选
- 按优先级筛选
- 按截止日期筛选
- 使用 JSON 文件持久化保存任务

工程要求：

- 代码结构清晰，函数职责明确
- 对非法优先级、非法日期、不存在的任务 id 等情况给出合理处理
- 至少编写 8 个测试用例
- 测试需要覆盖正常流程和异常输入

## 四、AI 协作要求

学生必须提交：

- 第一次请求 AI 生成代码的提示词
- AI 第一版输出摘要或完整输出
- 至少 3 轮后续交互记录
- 每轮交互说明：提示目标、AI 回应、发现的问题、自己的处理
- 最终反思报告：说明 AI 的问题、自己的修改、验证方法和个人贡献

## 五、固定提交格式

请将以下目录压缩为 zip：

```text
TaskFlow_Submission/
  README.md
  final/
    taskflow.py
  tests/
    test_taskflow.py
  ai/
    ai_initial_prompt.md
    ai_initial_output.md
    interaction_log.md
  report/
    reflection.md
```

## 六、评分标准

总分 100：

- 最终程序功能：25 分
- 代码质量与工程结构：15 分
- 测试与验证：15 分
- AI 协作过程记录：15 分
- 现场答辩理解表现：20 分
- 反思与原创性说明：10 分

## 七、答辩重点

答辩系统会围绕以下问题进行有限轮次追问：

- 学生是否理解最终代码的数据结构和关键函数
- 学生是否能指出 AI 初版代码的问题
- 学生是否能解释每轮提示的目标和处理结果
- 学生是否通过测试验证 AI 代码
- 学生是否能说明自己接受或拒绝 AI 建议的原因
- 学生是否能把个人贡献对应到具体代码、测试或文档
