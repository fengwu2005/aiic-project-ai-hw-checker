一、问题情境

你正在开发一个命令行图像处理工具。用户希望读取图片、保存图片，并对图片进行放大、缩小、旋转、 裁剪、反色、模糊、边缘提取和中值滤波等操作。作业重点是稳定的函数接口、像素处理逻辑、 参数校验、图像模式转换和可解释的验证方法。

二、任务目标

使用 Python 和 Pillow/PIL 实现命令行图像变换工具 ImageLab。 程序入口固定为 final/image_ops.py。

  • 普通用户可以通过命令行处理图片。
  • 课程系统可以通过固定函数接口进行隐藏验收。
  • 最终代码需要处理非法参数和损坏图片，不能静默吞掉问题。

三、核心交付

  • 一个可运行的 final/image_ops.py。
  • README.md：说明运行方式、依赖和功能清单。
  • report/report.md：说明实现方法、验证方式、关键贡献和已知不足。
  • 答辩时能解释图像函数、像素处理、隐藏验收和报告中的具体证据。

详细要求

必做图像功能

基础处理

  • 读取图片并转换成稳定可处理的图像模式。
  • 保存图片到目标路径。
  • 按比例放大和缩小图片。
  • 按角度旋转图片。
  • 按坐标裁剪图片。

像素与滤波

  • 反色处理。
  • 均值模糊。
  • 固定卷积核边缘提取。
  • 中值滤波。
  • 对非法参数和未知操作给出明确错误。
固定函数接口

final/image_ops.py 必须暴露以下函数名。CLI 可以自由设计，但不能破坏这些接口。

load_image(path) save_image(image, path) resize_image(image, scale) rotate_image(image, angle) crop_image(image, left, top, right, bottom) invert_image(image) blur_image(image) edge_detect(image) median_filter(image, size=3) transform_image(input_path, output_path, operation, **kwargs)

接口约束

  • 图像处理函数应返回新的 Pillow Image 对象。
  • load_image 应返回可稳定处理的图像对象，例如 RGB 图像。
  • save_image 必须实际写出目标文件。
  • resize_image 的 scale 必须大于 0。
  • crop_image 必须校验裁剪框在图像范围内。
  • median_filter 的 size 必须为大于 0 的奇数。
  • transform_image 至少支持 resize、rotate、crop、invert、blur、edge、median。
报告要求
  • 说明项目结构、运行方式和依赖。
  • 说明核心图像函数的实现方法，至少覆盖图像模式、裁剪校验、均值模糊、边缘检测和中值滤波。
  • 说明你用哪些输入进行自测，预期结果是什么，和隐藏验收可能检查什么对应。
  • 说明自己的关键实现、工程取舍和已知不足。
  • 报告中的说法应能对应到具体函数、行号、验收项或运行结果。

在完成全部必做功能之后，可以设计额外图像功能，例如灰度化、亮度/对比度调整、 自定义卷积核、批量处理目录图片、添加水印、直方图统计或多步流水线变换。只有这些自定义扩展可以作为 bonus。

系统不要求提交任何外部工具使用记录。它重点检查终版代码是否通过验收、报告是否能解释实现与验证， 以及学生在答辩中能否把回答落到具体函数、边界输入、图像概念和报告证据。
